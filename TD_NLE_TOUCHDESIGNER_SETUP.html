1. Overview & Philosophy

TD_NLE is a TouchDesigner-native modular NLE and sequencer. DAT tables are the single source of truth. Fixed shell COMPs load external .tox templates on demand. The UI reflects the model — never the other way around.

Core Rule Do not use Replicator COMP in the playback path. Use a fixed set of shell COMPs that load external .tox templates via Engine COMP. Replicator is reserved for admin/browser tools only.

No Replicators at Runtime

A fixed pool of shell COMPs (scene3d, scene2d, fx, video, audio) is created once. The timeline assigns clips to available shells by shell_type. Shells are never created or destroyed during playback.

Two-Tier Parameter Ownership

Templates own their native controls. The project owns only the overrides it cares about. Overrides live in overrides_json per clip and in overrides_table for inspector snapshots.

Separate Project & Template Save

.toe = full session. A JSON sidecar holds portable timeline state. .tox = individual template, saved explicitly by the author via RMB → Save Component. Never auto-saved by the timeline.

5. Shell Contracts

All scene shells (scene3d, scene2d, fx) use Engine COMP internally to isolate each .tox in a separate TouchEngine process. Video and audio shells use Movie File In TOP and Audio File In CHOP respectively.

Engine COMP Key Points Clock Mode = Synchronized. Load on Demand = On. Reload on Crash = On. Custom parameters on the .tox surface as Engine COMP parameters — parameter direction is host → engine only. onStart()/onExit() never fire under TouchEngine; use onCreate() for init.

Required Custom Parameters — All Shells

Par NameTypeRequiredDescription
ActivetoggleYesEnable/disable cooking
BypassrendertoggleYesSkip render when off-screen (opacity=0)
LocaltimefloatYesCurrent clip time in seconds
ClipprogressfloatYes0–1 normalized progress within clip
OpacityfloatYesMaster opacity (0–1)
SeedintYesRandomization seed
PrewarmtoggleOptionalPre-warm on preroll before clip start
AudioReactivefloatOptionalAudio reactivity amount (0–1)
BlendHintmenuOptionalSuggested blend mode for Layer Mix
OUT_TOPNull TOP outputYesFinal visual output. Black for audio clips.

scene3d

3D generative scenes. Engine COMP loads .tox. Internal network: Geometry, Render TOP, Camera COMP, Light COMP → OUT_TOP.

Optional extra pars: Camera (str), Geometry (str).

scene2d

2D comp / GLSL / generative. Engine COMP loads .tox. Internal network: any 2D pipeline → OUT_TOP.

Optional extra pars: Resolution (menu).

fx

Effect processors that take an input TOP. Internal network: in1 wired input or InputPath (str) → effect chain → OUT_TOP.

Required extra pars: Amount (float 0–1), Mode (menu).

Shell Save / Reload Workflow

  • Save Template: RMB → Save Component on the loaded external .tox inside the shell. Never triggered automatically by the timeline.
  • Reload Template: Pulse the Enable External .tox parameter on the shell's Engine COMP, or call extPlayerPool._reloadShell(slot_index).
  • Open Network: See Section 12 — uses ui.panes.createFloating().

All six extensions are promoted on /timeline_system. Each lives in scripts/ as a .py file — paste into a Text DAT, set as extension, promote.

7. Build Walkthrough

Build in this order. Each step is independently testable.

Step 1 — Model DATs

  1. Inside /timeline_system, create a Base COMP named model.
  2. For each table in model/*.txt: create a File DAT, point it at the .txt file, set Sync to File → Load on Start. Name each DAT to match its file (clips_table, tracks_table, etc.).
  3. For runtime editing, replace File DATs with Table DATs initialized from the schema headers. File DATs are read-only.
  4. Verify: op('/timeline_system/model/clips_table').numCols → 23

Step 2 — Transport

  1. Create container transport inside /timeline_system.
  2. Add a Time COMP named local. Set Clock Source → Timeline.
  3. Add a Timeline CHOP named timeline_master. Connect Time COMP as input.
  4. Add a Timecode CHOP named timecode_master. Connect Timeline CHOP.
  5. Add a Text DAT with extTransport.py contents. On /timeline_system Extensions page, add it as an extension, Promote = On.
  6. Verify: op('/timeline_system').ext.extTransport.CurrentFrame() returns current frame number.

Step 3 — Player Shells

  1. Create container players inside /timeline_system. Inside it, create sub-containers: scene3d, scene2d, fx, media.
  2. For each scene3d shell (start with 2: scene3d_shell_1, scene3d_shell_2):
    • Inside the shell COMP, add an Engine COMP named engine_comp. Set Clock Mode = Synchronized, Load on Demand = On, Reload on Crash = On.
    • Add a Switch TOP named out_select_top.
    • Add a Null TOP named OUT_TOP. Wire: engine_comp output → Switch TOP input 0 → OUT_TOP.
    • Add custom parameters to the shell COMP: Active (toggle), Bypassrender (toggle), Localtime (float), Clipprogress (float), Opacity (float), Seed (int).
  3. Repeat for scene2d and fx shells (same structure).
  4. For each media slot (2 video + 2 audio minimum):
    • Video slot: Engine COMP (off) + Movie File In TOP named movie_filein (Play Mode = Specify Index, Index Unit = Seconds) + Switch TOP + Null OUT_TOP.
    • Audio slot: Audio File In CHOP named audio_filein (Play Mode = Specify Index, Index Unit = Seconds). No visual output — audio goes to mix bus.

Step 4 — Wire All Extensions

  1. Create Text DATs in scripts/ for each extension. Paste content from the repo .py files.
  2. On /timeline_system Extensions page, add all 6 in this order: extTransport, extPlayerPool, extSequencer, extAnalysis, extRecorder, extIngest. Set Promote = On for all.
  3. Add a CHOP Execute DAT wired to the Timeline CHOP output. In its onValueChange (or use a DAT Execute on the per-frame cook): call op('/timeline_system').ext.extSequencer.Tick().
  4. Verify: op('/timeline_system').ext.extPlayerPool.PoolSize returns your total shell count.

Step 5 — Ingest & Browse

  1. The extIngest extension is already wired (Step 4). Test drag-and-drop by enabling the Drop Files panel parameter on the clip_canvas container.
  2. Create a Text DAT named clip_canvas_cbs. Paste clip_canvas_callbacks.py. Wire it as the Callbacks DAT parameter of clip_canvas.
  3. Create a File DAT pointing to editor/media_bin.txt. This DAT drives the Browse button rows — one button per row, each calling clip_canvas_callbacks.BrowseTox(shell_type=row['shell_type']).
  4. Set project_settings['template_root','value'] to your templates directory. The file picker will open there by default.
  5. Test: op('/timeline_system').ext.extIngest.BrowseAndIngest(shell_type='scene3d') → file picker opens → choose a .tox → row appears in clips_table with correct shell_type.

Step 6 — Layer Mix TOP

  1. Create container mix. Add a Layer Mix TOP named layermix_main.
  2. Wire all shell OUT_TOP outputs as inputs to layermix_main (one per visual slot).
  3. Add a Null TOP named program_out wired from layermix_main. Add another named record_out.
  4. extSequencer._updateLayerMix() drives each layer's Opacity and Blend parameters automatically each frame.

Step 7 — Editor UI

  1. Create container editor. Import editor/editor_state.txt as a Table DAT (key/value pairs).
  2. Import editor/transport_controls.txt and editor/media_bin.txt as Table DATs for button layouts.
  3. Build Container COMPs for: transport_bar, timeline_ruler, clip_canvas, note_canvas, inspector, track_headers.
  4. Wire callback DATs: clip_canvas_callbacks.py → clip_canvas; timeline_ruler_callbacks.py → timeline_ruler; transport_bar_actions.py → transport_bar buttons.
  5. All viewport math (zoom, pan, pixel↔frame, track↔y) is handled by editor_state_utils.py — call functions from Panel Execute DATs or button callbacks.

8. Media Ingest & Browse

Two paths to add clips: drag-and-drop onto the canvas, or click a Browse button for a specific shell type.

Drag and Drop

  1. Enable Drop Files = On on the clip_canvas Container COMP (Panel page).
  2. Wire dropzone_callbacks.py as the Callbacks DAT.
  3. Drop any supported file — .tox, .mp4, .mov, .wav, .aif, etc.
  4. extIngest._detectShellType() infers shell type from folder path conventions:
    /fx/ or /fx_ or effect → fx
    /2d/ or scene2d or /comp/ → scene2d
    everything else → scene3d (default)

Browse Buttons (media_bin.txt)

Each row in editor/media_bin.txt drives one button:

Override Shell Type After Ingest

The inspector can change shell_type on any existing clip:

op('/timeline_system').ext.extIngest.SetClipShellType('1', 'scene2d')

Or from the inspector callback: clip_canvas_callbacks.ChangeShellType(clip_id, shell_type)

Nondestructive Trim

Source files are never modified. source_in and source_out are stored in the clip row as frame offsets. At playback, extPlayerPool._driveMediaPlayer() converts to seconds and sets movie_filein.par.index or audio_filein.par.index.

# video: index is seconds from source start
source_seconds = source_in_s + local_frame / project_fps

# audio: same conversion (AudioFileIn also uses seconds with indexunit='seconds')
source_seconds = source_in_s + local_frame / project_fps

9. TDMorph Integration

TDMorph PresetManager.tox lives at /timeline_system/scripts/preset_manager, targeting custom parameters on scene shells. All preset actions are triggered from notes_table event rows.

Placement

  1. Copy TDMorph/PresetManager/PresetManager.tox into scripts/.
  2. In TD, drag it into /timeline_system/scripts/ as a Base COMP named preset_manager.
  3. Point PresetManager's target at the shell COMP custom parameters (Active, Localtime, Opacity, Seed, etc. on player_1..N).

Python API

pm = op('/timeline_system/scripts/preset_manager')
pm.StorePreset('scene_a')                                   # save current state
pm.JumpToPreset('scene_a')                                  # instant — use on clip load
pm.MorphPreset('scene_a', morphTime=1.5, morphCurve='easeIn')  # smooth morph
pm.RandomMorph(mode='Uniform')                              # generative
pm.PresetsSequence(keysSequence=['a', 'b', 'c'])            # sequence

notes_table Event Types

event_typetargetvalueparams_json
morph_presetop path to PresetManagerpreset name{"morphTime": 1.5, "morphCurve": "easeIn"}
jump_presetop pathpreset name—
random_morphop pathmode (e.g. Uniform)—
preset_sequenceop pathcomma-separated preset names—

Patterns Library (algorithmic note generation)

SuperCollider-style generators inside PresetManager.tox — generate note rows algorithmically:

  • Pseq(seq, repeats) — ordered sequence
  • Prand(seq, repeats) — random pick
  • Pxrand(seq, repeats) — random, no consecutive repeats
  • Pwrand(seq, weights, repeats) — weighted random
  • Pbrown(lo, hi, step) — brownian walk (good for morph times)
  • Pwhite(lo, hi) — uniform random float
PresetAnimator → Automation Lanes PresetAnimator.CreateAnimation() bakes presets into an Animation COMP. That feeds the existing lane_{id} Keyframe CHOP pipeline in extSequencer._sampleAutomation().

10. EssentiaTD Integration

Five C++ CHOP plugins for batch audio analysis. Primary workflow is offline — analyze once, cache results to DATs, use during playback with zero overhead.

CHOP Plugin Suite

PluginOutput ChannelsUse Case
Essentia SpectrumFrequency spectrum binsVisual frequency response
Essentia Spectralcentroid, spread, flux, rolloff, contrastTimbral descriptors
Essentia Tonalkey, key_scale, key_strength, chordsHarmonic analysis → analysis_table
Essentia Rhythmbpm, beat_phase (sawtooth 0→1), beat_confidenceBPM detection, beat tracking → markers
Essentia Loudnessloudness_db, loudness_range, momentary, integratedDynamic range analysis

Batch Workflow

  1. User calls extAnalysis.Analyze(path) (or drops audio onto a designated analysis zone).
  2. EssentiaTD CHOPs run in a background thread — file loaded, all CHOPs cook once in batch mode (par.timeslice = False).
  3. Results harvested: _harvestRhythm(), _harvestLoudness(), _harvestTonal().
  4. Written to model/analysis_table (per-frame) and model/markers_table (beat/phrase positions).
  5. Fallback chain: EssentiaTD → Palette audioAnalysis → manual markers.

beat_phase for Visual Sync

# Essentia Rhythm CHOP outputs beat_phase as a sawtooth 0.0 → 1.0 per beat
beat_phase_val = op('essentia_rhythm')['beat_phase'][0]   # 0.0 to 1.0 this beat
# Use as modulation source for shader uniforms or TDMorph morph blend

analysis_table → notes_table (suggested workflow)

After analysis, the beat/onset positions in markers_table can be converted to notes_table rows (event_type = trigger or morph_preset) to sync visual events to detected beats.

11. Project & Template Save

Key Principle Project save captures references and overrides, never scene internals. Template save is deliberate and local to the template being authored.

Save Project

Captures: clip rows, tracks, notes, markers, transport settings, selected templates, parameter overrides.

  • .toe — full TouchDesigner session (networks, wiring)
  • project.json sidecar — portable timeline state. Written by extIO.SaveProject(path) using File Out DAT or Python json.dumps(). JSON DAT can re-import it.
extIO.SaveProject('project.json')
extIO.LoadProject('project.json')

Save Template

Author edits the loaded .tox inside a shell in the TD network editor, then saves explicitly:

  • RMB → Save Component on the loaded sub-COMP inside the Engine COMP
  • Or via Python: comp.save(path)
  • Never auto-saved when the timeline changes — templates are permanent assets

Reload Template

# Pulse the Engine COMP's external tox reload
engine = op('/timeline_system/players/scene3d/scene3d_shell_1/engine_comp')
engine.par.reloadpulse.pulse()
# or via extPlayerPool:
ext.extPlayerPool._reloadShell(slot_index)

IO Subsystem (/timeline_system/io/)

The io/ container handles:

  • Folder DAT — scans templates/scene3d/, templates/scene2d/, templates/fx/ and populates template_catalog DAT. Set Active = On, Include Subfolders = Off, filter by .tox extension.
  • JSON DAT — parses loaded project.json sidecar into a table (Output Format = Table, list-of-dicts → one row per clip).
  • File Out DAT — writes DAT contents to disk on save. Set par.file to the sidecar path, pulse par.write.
# Folder DAT output columns (auto):
# Name, Base Name, Extension, Type, Size, Depth, Folder, Path, Relative Path, Date Modified…
# shell_type is derived by the catalog scanner from the subfolder name

12. Open Network Button

The inspector can open the currently loaded template in a floating network editor pane — enabling live in-place editing without leaving the timeline flow.

def OpenNetwork(comp_path):
    """Open a floating Network Editor pane pointing at the given COMP."""
    target = op(comp_path)
    if target is None:
        return
    pane = ui.panes.createFloating(
        type=PaneType.NETWORKEDITOR,
        name='TD_NLE — ' + target.name,
        maxWidth=1600,
        maxHeight=900,
    )
    pane.owner = target

Call this from the inspector's Open Network button with the path to the Engine COMP's loaded sub-component:

# Example: open the scene currently loaded in shell 1
shell = op('/timeline_system/players/scene3d/scene3d_shell_1')
engine = shell.op('engine_comp')
# Point at the loaded .tox network inside the engine process
OpenNetwork(engine.path)
Panes API Reference ui.panes.current — active pane. pane.owner — sets the COMP the pane points to. ui.panes.createFloating(type, name, maxWidth, maxHeight) — returns a Pane object.

13. Playback Rules

These rules run every frame inside extSequencer.Tick().

  1. Resolve current frame f from extTransport.CurrentFrame().
  2. Build track filter from tracks_table mute/solo state.
  3. extPlayerPool.Update(f, clip_filter) — assigns active clips to shells by shell_type, drives parameters, handles preroll and unload.
  4. _evaluateNotes(f) — finds all notes where start_frame ≤ f < end_frame. Fires _fireNote() on the frame equal to start_frame.
  5. _sampleAutomation(f) — reads lane_{id} Keyframe CHOPs, writes parameter values to target OPs.
  6. _updateLayerMix() — sets Opacity and Blend on each Layer Mix TOP layer from active assignments.
  7. If recorder is active, call extRecorder.UpdateBurnin(f).

Shell Assignment Logic

extPlayerPool maps shell_type to a shell pool. Within a pool, it finds the first free slot. If no free slot exists, it logs and skips (increase shell count to raise polyphony).

shell_typePoolFallback
scene3dplayers/scene3d/none — increase shell count
scene2dplayers/scene2d/none
fxplayers/fx/none
videoplayers/media/ (video slots)none
audioplayers/media/ (audio slots)none

Preroll

If a clip has preroll_frames > 0, the shell is loaded and Prewarm = True is set before the clip start frame. This warms the GPU resources so the clip is ready exactly at start. Preroll clips do not appear in Layer Mix output (Opacity = 0).
