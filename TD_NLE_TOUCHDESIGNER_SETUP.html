1. Overview

TD_NLE is a TouchDesigner-native modular timeline and sequencer where DAT tables act as the edit decision list, transport resolves every time mode back to internal frames, and reusable players load either scene clips or indexed media clips.

Current Canonical Choice This guide supersedes older notes where the live UI container was called /ui. The current runtime layout lives under /timeline_system/editor.
Reference Philosophy The interaction feel is inspired by Zerrorâ€™s timeline behavior, but the backend is fully custom: no dependency, no runtime integration, and no reuse of preset-timeline internals.

What This Guide Builds

  • A blank-network assembly path for /timeline_system.
  • Root extensions for transport, player pooling, sequencing, analysis, recording, and ingest.
  • Model DATs using the current repo schemas.
  • Editor scaffolding for transport, ruler, clip canvas, note canvas, automation canvas, inspector, and media bin.
  • Scene-player and media-player pools that match the current extPlayerPool expectations.
  • A recordable visual path and a preferred audio_mix audio bus.

Out of Scope for This Pass

  • Fully finished clip, note, and automation panel interactions.
  • Embedded movie-audio extraction from video clips.
  • Final timecode chase polish and advanced preroll behavior.
  • Any locked third-party timeline component.
  • Automatic .toe rewiring; this guide describes the intended network you still need to build in TouchDesigner.

4.6 Build the Transport Subsystem

The current transport is frame-first internally, but it accepts external seconds cleanly. Build the following under /timeline_system/transport.

NodeTypeCanonical Setup
localBase COMPContainer for the Time COMP so the path remains transport/local/time.
local/timeTime COMPSet it up in frames, with rate matching project FPS. extTransport toggles its play state and writes range start values directly.
timeline_masterInfo CHOP or equivalent CHOPTarget local/time and expose a channel named frame.
timecode_masterCHOPCan start as a Constant CHOP with an index channel. Later swap to LTC/MTC or SMPTE ingest.
external_time_inCHOPPublic interface is seconds. Use a seconds channel if possible. extTransport also accepts time, sec, or the first channel.
transport_resolveCHOP Execute DATWatch the frame-driving CHOP and tick the sequencer on frame changes.
Current Canonical Choice Transport supports internal, external, timecode, beat, and scrub. External input arrives in seconds and is converted to internal frames immediately.

Baseline transport callback:

# /timeline_system/transport/transport_resolve (CHOP Execute DAT)
def onValueChange(channel, sampleIndex, val, prev):
    if channel.name == 'frame':
        op('/timeline_system').ext.extSequencer.Tick()

If you later add a dedicated resolved-frame CHOP, repoint the CHOP Execute DAT there. For the baseline build, watching timeline_master is enough to prove the transport path.

Verify Now Set clock_source to internal, hit play on local/time, and confirm extTransport.CurrentFrame() advances. Then set clock_source to external, feed a test value into external_time_in, and confirm the returned frame changes accordingly.

6. Playback Rules

Internal Frame Is Truth

Every playback mode resolves to an internal frame count. UI panels, external clocks, timecode, and beat logic all collapse back to that one engine truth.

External Time Arrives in Seconds

external_time_in is a seconds-facing input. extTransport converts it to frames immediately and applies loop logic afterward.

Scene Clips Stay in Scene Players

.tox clips are loaded into the Engine COMP scene pool and use the scene clip contract: Active, Localtime, Clipprogress, Opacity, and preroll toggles.

Video and Audio Stay in Media Players

Movie and audio files are driven by seconds index. They are not forced through scene-player wrappers.

Visual vs Audio Separation Matters

.tox and video clips contribute visual layers. Audio clips route to the audio bus only and should never consume fake visual mix layers.

Trimming Is Nondestructive

source_in, source_out, in_offset, and stretch behavior only change DAT values and playback math. The source media files are untouched.

8. Validation Checkpoints

Use this final checklist after the network is assembled. If one step fails, do not move deeper into recording or timecode until the earlier path is clean.

Final Integration Checklist

  • Drop one .tox clip onto clip_canvas and confirm a new clips_table row appears with media_type = tox, source_in = 0, source_out = -1, and fps = 0.
  • Drop one video clip and confirm it writes media_type = video, source_out = -1, and a usable fps value.
  • Drop one audio clip and confirm it writes media_type = audio with nondestructive source trim values.
  • Drop multiple files in one action and confirm each subsequent row starts after the previous clip's duration.
  • Wire native Material Design Icon glyphs into transport_bar and confirm they remain legible at both normal transport sizes and compact track-header densities.
  • Call the new zoom, pan, playhead-drag, and Home-to-fit helpers and confirm only editor_state changes while clip/media data stays in model DATs.
  • Hover and select clips on clip_canvas and confirm hover_frame, hover_track, hover_clip_id, and selected_clip_ids update in editor_state.
  • Wire the GLSL overlays declared in glsl_panels for timeline_ruler and clip_canvas and confirm playhead, grid, and selection overlays line up exactly with DAT-driven frame positions.
  • Bypass the GLSL overlay COMPs and confirm transport controls, drag/drop ingest, and callback-driven selection still work through native panels.
  • Switch clock_source between internal, external, and timecode and verify the current frame remains coherent.
  • Record a short review pass and confirm moviefileout1 receives the visual branch plus the audio_mix CHOP.
  • Confirm the build works without any dependency on Zerror assets, preset timeline APIs, or locked third-party components.
Reality Check The repo now contains the extension code, editor UI helper scripts, theme/icon DAT seeds, and placeholder node names, but it does not automatically wire the actual .toe network for you. This guide remains the canonical wiring target for that manual TouchDesigner step.

9. Known Gaps and Next Phases

Note Editing UI

note_canvas exists as a target, but direct note creation and editing panels are still not built.

Automation UI

automation_canvas and automation_table are part of the model, but the full keyframe editor still belongs to a later pass.

Actual .toe Wiring

The repo structure now documents the intended network, but the live TouchDesigner project still needs to be wired to match it node-for-node.

Embedded Movie Audio

Video clips are currently treated as visual clips. Standalone audio clips handle the audio lane; embedded movie audio remains deferred.

Timecode Polish

Advanced SMPTE chase behavior, preroll nuance, and cleaner burn-in metadata still belong to a later phase.

Trim and Stretch Gestures

The new helper layer now covers playhead drag, zoom, pan, selection, and clip move helpers. Trim handles, stretch gestures, and their final GLSL/native visuals still need live panel wiring.
